Judgment JSON Pipeline
Offline · Ollama

Process a judgment PDF

Uploads a PDF to the local pipeline API and returns structured JSON.

Output JSON

API expectation

Frontend expects a local endpoint /pipeline/process that accepts multipart/form-data with fields file (PDF) and model (string). It should respond with the structured JSON shown below.

{
  "case_name": "",
  "citation": "",
  "court": "",
  "date": "",
  "bench": [],
  "facts": "",
  "procedural_history": "",
  "issues": [],
  "arguments": {
    "petitioner": "",
    "respondent": ""
  },
  "judgment_summary": "",
  "reasoning": "",
  "ratio": "",
  "legal_principles": [],
  "citations": [],
  "final_order": ""
}
